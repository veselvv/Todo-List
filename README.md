# Todo-List (Task Manager)


TUI менеджер задач, написанный на **C** с использованием **PostgreSQL** для постоянного хранения данных. Проект позволяет создавать задачи, устанавливать дедлайны и сохранять прогресс между сессиями.

## ✨ Основные возможности

*   **Добавление задач**: Создание новых задач с описанием.
*   **Учёт сроков**: Возможность установки **дедлайна** (deadline) для каждой задачи.
*   **Просмотр списка**: Отображение всех активных и выполненных задач.
*   **Управление статусом**: Отметка статуса задач.
*   **Управление степенью важнности**: приоритет задач. 
*   **Персистентность**: Все данные сохраняются в базу данных PostgreSQL и загружаются при повторном запуске.

## 🛠 Технологический стек

*   **Язык**: C
*   **База данных**: PostgreSQL 15 (в Docker-контейнере)
*   **Сборка**: Makefile
*   **Окружение**: Linux

## 📦 Установка и запуск

### Предварительные требования

*   Компилятор **GCC** и утилита **Make**.
*   **Docker**.
*   Библиотека разработки для PostgreSQL (`libpq-dev`).

**Установка зависимостей на Ubuntu/Debian:**
```bash
sudo apt update
sudo apt install build-essential libpq-dev
# Установка Docker
sudo apt install docker.io
sudo systemctl enable --now docker
# Добавление текущего пользователя в группу docker (чтобы не использовать sudo)
sudo usermod -aG docker $USER
# Перезайди в сессию или выполни: newgrp docker
```
**Настройка базы данных (Docker)**

Проект рассчитан на работу с PostgreSQL, запущенным в Docker-контейнере со следующими параметрами:

| Параметр | Значение |
|:---------|:---------|
| Порт хоста | `5433` |
| Порт контейнера | `5432` |
| База данных | `users` |
| Пользователь | `postgres` |
| Пароль | `123` |
| Volume | `postgres_data` |

Проверка подключения
```bash
docker exec -it postgres_db psql -U postgres -d users
```

**Компиляция**

Проект содержит Makefile для удобной сборки:
```bash
make
```
**Запуск**

Убедись, что Docker-контейнер с PostgreSQL запущен (docker ps), затем выполни:
```bash
./todo_app
```
Или используй скрипт run.sh (он может автоматически поднимать контейнер):

```bash
chmod +x run.sh
./run.sh
```
